## 摘要

林纳斯·贝内迪克特·托瓦兹（Linus Benedict Torvalds）是一位芬兰裔美国软件工程师，他创造并持续领导着 Linux 内核的开发，Linux 内核是 Linux 操作系统（Linux operating system）的基础。托瓦兹于 1969 年出生在赫尔辛基，1991 年在大学期间开始开发 Linux，引发了一场改变技术格局的全球开源革命。他还创建了 Git，一种分布式版本控制系统，彻底改变了协作软件开发。他务实、精英主义的软件开发方法使他成为计算史上最具影响力的人物之一，尽管他粗暴的沟通方式也引起了广泛争议。

## 早年生活和背景

林纳斯·贝内迪克特·托瓦兹于 1969 年 12 月 28 日出生在芬兰赫尔辛基，父母是记者安娜（Anna）和尼尔斯·托瓦兹（Nils Torvalds）。[^1] 他的家人属于瑞典语少数民族，约占芬兰人口的 5%。[^1] 托瓦兹以美国诺贝尔奖得主、化学家莱纳斯·鲍林（Linus Pauling）的名字命名，来自一个知识分子家庭——他的祖父利奥·托恩奎斯特（Leo Törnqvist）是统计学家，另一位祖父奥勒·托瓦兹（Ole Torvalds）是诗人。[^1][^2]

托瓦兹的父母在 20 世纪 60 年代是赫尔辛基大学的校园激进分子，营造了一种重视思想独立和批判性思维的家庭氛围。[^1] 他进入计算机领域始于 20 世纪 80 年代初，当时他的祖父购买了一台 Commodore VIC-20 家用电脑。这次购买至关重要——11 岁的托瓦兹对编程产生了浓厚兴趣，花费无数时间编写代码并探索机器的功能。[^1][^2]

年轻的托瓦兹在编程方面展现出卓越的天赋，常常将计算机工作置于社交活动之上。这种早期的计算沉浸确立了定义他职业生涯的模式：注重实际问题解决、亲手实验以及偏好与机器打交道而非应对社交动态。他在芬兰瑞典语社区的童年也让他拥有了多元文化的视角，他在一个以芬兰语为主的国家长大，却能流利地说瑞典语。

知识渊博的家族传统、早期接触计算技术以及个人对技术挑战的倾向，为托瓦兹后来的成就奠定了基础。他祖父对那台 Commodore VIC-20 的投资无意中引发了一系列重塑全球计算的事件。

## 声名鹊起

托瓦兹于 1988 年至 1996 年在赫尔辛基大学学习，在 NODES 研究小组获得了计算机科学硕士学位。[^1] 他的学术生涯在第一年因夏季加入芬兰海军尼兰德旅（Nyland Brigade）以履行芬兰强制兵役而中断。托瓦兹选择了为期 11 个月的军官培训项目，并获得少尉军衔，担任炮兵观察员。[^1]

1990 年重返大学后，托瓦兹第一次接触到 Unix，形式是一台运行 ULTRIX 的 DEC MicroVAX。这次接触具有变革性。[^1] 他购买了计算机科学教授安德鲁·塔能鲍姆（Andrew Tanenbaum）的著作《操作系统：设计与实现》（Operating Systems: Design and Implementation），书中讨论了 MINIX，这是一个为教育目的设计的小型类 Unix 操作系统。1991 年 1 月 5 日，托瓦兹购买了一台基于 Intel 80386 的 IBM PC 克隆机，在收到一份 MINIX 副本后，他开始着手开发后来成为 Linux 的项目。[^1][^3]

从 1991 年 4 月到 8 月，21 岁的托瓦兹编写了最初的 Linux 内核。[^3] 他从 Intel 80386 汇编语言的任务切换器和终端驱动程序开始，专门为他的硬件编写程序，独立于完整的操作系统。[^3] 1991 年 8 月 25 日，托瓦兹向 comp.os.minix 新闻组发表了他著名的声明，将他的项目描述为 386/486 计算机的爱好操作系统。[^3]

1991 年 9 月 17 日，托瓦兹准备了 Linux 的 0.01 版本，并将其上传到芬兰大学和研究网络（FUNET）的 FTP 服务器。[^3] 这个第一个版本包含 10,239 行代码。[^3] 1991 年 10 月 5 日，他宣布了第一个“官方”版本，0.02。[^3] 关键的突破发生在 1991 年 12 月，版本 0.11 是第一个可以自举（在运行 Linux 内核的计算机上编译）的版本。[^3]

有趣的是，托瓦兹最初想将他的创造命名为“Freax”，这是“free”（自由）、“freak”（怪胎）和“x”（暗指 Unix）的混合词。他曾考虑过“Linux”，但认为它过于自负而放弃了。[^3] 赫尔辛基技术大学的 FTP 服务器管理员之一阿里·莱姆克（Ari Lemmke）不喜欢“Freax”，未经托瓦兹同意就将项目命名为“Linux”。后来，托瓦兹还是同意了这个名字。[^3]

## 主要成就

### Linux 内核

托瓦兹的主要成就是自 1991 年以来创建并维护 Linux 内核。[^8] 起初是一个学生项目，后来发展成为一个为全球计算基础设施提供动力操作系统的基础。Linux 已成为互联网的支柱，在全球服务器上运行，并作为 Android 智能手机、超级计算机、嵌入式系统和无数其他设备的操作系统。[^8]

Linux 的成功不仅源于其技术优势，还源于托瓦兹将其作为开源发布的决定。这一选择不仅启动了一个项目——它催生了现代开源运动。[^8] 通过允许任何人自由使用、修改和分发代码，托瓦兹催生了一种前所未有的协作开发模式。全球数千名开发人员为 Linux 贡献了力量，创造了人类历史上最大的协作项目之一。

托瓦兹的发展理念强调务实而非意识形态。正如他所说，开源“不是一场战斗——它只是一种更好的合作方式和代码生成方式。”[^7] 这种实用的方法吸引了专注于解决实际问题而非追求抽象原则的贡献者。他认为，“开源最重要的部分是人们可以做他们擅长的事情。”[^7]

### Git 的创建

2005 年，托瓦兹创造了分布式版本控制系统 Git，做出了另一项开创性贡献。[^4] Linux 内核开发社区一直使用 BitKeeper，一个专有工具，但当与该商业开发商的关系破裂后，托瓦兹需要一个替代品。[^4]

托瓦兹没有采用现有系统，而是仅用了十天就编写了 Git。[^4] 他在 2005 年 4 月 7 日对 Git 进行了第一次提交。[^4] 项目启动几天后，Git 就完全实现了自举，几周内，它就可以支持 Linux 内核开发。[^4] 托瓦兹的主要目标是分发性、高性能和绝对可靠性，以捕获任何损坏。[^4]

Git 彻底改变了全球的软件开发。托瓦兹维护了 Git 约四个月，然后将维护权移交给了从开发第一周就开始参与的琼尼奥·哈马诺（Junio Hamano）。[^4] 今天，Git 是占主导地位的版本控制系统，被全球数百万开发人员和组织使用。GitHub、GitLab 和 Bitbucket 等平台围绕 Git 构建了完整的生态系统，从根本上改变了软件的开发和共享方式。

### 奖项和认可

托瓦兹的贡献赢得了广泛认可。他获得了 2012 年千禧技术奖，以表彰他“创造了一个新的计算机开源操作系统，从而诞生了被广泛使用的 Linux 内核”。[^6] 其他主要奖项包括 2014 年 IEEE 计算机学会计算机先驱奖、2018 年 IEEE 颁发的 Masaru Ibuka 消费电子奖、NEC 的 C&C 奖（2010 年）、武田社会/经济福祉奖（2008 年）、英国计算机学会的 Lovelace 奖章（2000 年）以及电子前线基金会先驱奖（1998 年）。[^6]

他于 2012 年被选入互联网名人堂（Internet Hall of Fame），并于 2008 年被选入计算机历史博物馆名人堂（Computer History Museum's Hall of Fellows）。[^6] 《时代》杂志在 2000 年将其评为“时代百人：20 世纪最重要人物”的第 17 位，并在 2004 年被评为世界上最有影响力的人物之一。[^6]

### 其他贡献

除了 Linux 和 Git，托瓦兹还于 2011 年底开始创建 Subsurface，一个用于记录和规划水肺潜水的软件。[^4] 这个项目虽然不如他其他工作那样引人注目，但它展示了他持续亲手开发软件的方法，以及他愿意为个人需求构建工具的意愿。

## 争议和批评

### 粗暴的沟通方式

托瓦兹的管理风格在他的职业生涯中引起了相当大的争议。他在 Linux 内核邮件列表（LKML）上出了名的粗暴，经常对他认为有问题代码——以及人——发表充满脏话的训斥。[^5] 在一次会议上，托瓦兹表示：“我不是一个好人，我不在乎你。我关心技术和内核——那对我来说才是重要的。”[^5]

他严厉的沟通方式给许多开发人员造成了有毒的氛围。著名的 Linux 开发人员莎拉·夏普（Sarah Sharp）公开批评托瓦兹“口无遮拦和残暴的贬低”，警告说她“受够了”并恳求：“你不需要大喊大叫，给我起外号，或让我闭嘴！”[^5] 另一位开发人员指出：“他在新来者面前斥责资深开发人员。这不太可能鼓励人们成为资深开发人员。”[^5]

托瓦兹营造的文化甚至超越了他本人——Linux 内核维护者也经常变得直言不讳、无礼或粗暴，顶级开发人员经常互相大喊以纠正行为。[^5] 虽然有些人辩解说这是为了保持高标准，但其他人认为这会制造进入障碍，并赶走发现环境不受欢迎的有才华的贡献者。

### 2018 年道歉和行为准则

2018 年 9 月中旬，托瓦兹宣布他将休假，“以获得一些关于如何理解他人情感并做出适当回应的帮助”。[^5] 这一前所未有的举动是在多年来关于他行为的批评日益增多的背景下发生的。与此同时，Linux 内核项目通过了一项新的行为准则，旨在使社区更具包容性，并消除侮辱性、诽谤性的言论和行为。[^5]

该行为准则取代了 2015 年的一项名为“冲突守则”（Code of Conflict）的规定，该规定曾简单地建议人们“要善待彼此”。[^5] 然而，新守则引发了巨大的反弹。一些程序员认为这是对精英主义的攻击，而另一些人则称这些攻击是别有用心的厌女症。[^5] 有些人担心该守则可能因缺乏政治正确性而终结他们作为内核开发者的任期，而另一些人则认为它没有足够的执行机制来带来有意义的改变。[^5]

### 对多样性和包容性的影响

托瓦兹的攻击性风格以及由此产生的文化因抑制内核开发的多样性而受到批评。Linux 内核社区历史上绝大多数是男性，主要是白人和亚洲人，女性和代表性不足的少数族裔的参与度很低。批评者认为，托瓦兹确立并容忍的敌意沟通规范对这种缺乏多样性有显著贡献。

这场争议凸显了不同精英主义愿景之间的紧张关系：一种只关注代码质量，而不顾人们如何被对待；另一种则认为包容性的环境对于获取最广泛的人才库是必要的。托瓦兹最终承认他需要改变，这表明他认识到他以前的方法是有代价的，尽管人们对后续的改变是否足够有不同看法。

## 个人生活

林纳斯·托瓦兹于 20 世纪 90 年代中期与六次芬兰国家空手道冠军托维·托瓦兹（Tove Torvalds，娘家姓 Monni）结婚。[^9] 他们于 1993 年秋季首次相遇，当时托瓦兹正在赫尔辛基大学为学生举办入门级计算机实验课程。他指示课程参与者给他发一封电子邮件进行测试，托维回复了一封要求约会的电子邮件。[^9]

这对夫妇育有三个女儿：帕特里夏·米兰达（Patricia Miranda，1996 年出生）、丹妮拉·约兰达（Daniela Yolanda，1998 年出生）和赛勒斯特·阿曼达（Celeste Amanda，2000 年出生），其中两人在美国出生。[^9] Linux 内核的 reboot 系统调用接受她们的出生日期（以十六进制表示）作为魔术值，这是一种独特的、永久嵌入代码中的技术人格的俏皮致敬。[^9]

托瓦兹于 1997 年移居美国，并与家人定居。[^9] 他于 2010 年成为美国公民，同时保留芬兰国籍。[^9] 该家庭一直居住在俄勒冈州波特兰，托瓦兹在那里继续从事 Linux 内核的工作。

除了编程，托瓦兹还对水肺潜水表现出兴趣，这促使他开发了 Subsurface 潜水日志软件。[^4] 这种爱好表明他倾向于通过软件开发来解决个人需求——当现有的潜水日志软件不符合他的要求时，他自己构建了一个。

托瓦兹将自己描述为务实而非富有远见：“我不是一个富有远见的人。我是一名工程师。我乐于那些四处张望看星星的人，但我正在观察地面，我想在摔倒之前修好那个坑。”[^7] 这种自我评价反映了他务实、解决问题的导向，以及对具体改进而非抽象理论的偏好。

他的人格结合了对技术卓越的强烈关注，以及在应对社交和情感动态方面的明显不适。尽管在 2018 年的争议之后，他努力改善了自己的沟通方式，但同事们注意到，他根本的关注点仍然是代码质量和技术价值，而不是人际关系。

## 遗产

### 转变计算基础设施

林纳斯·托瓦兹对全球计算基础设施的影响怎么强调都不为过。Linux 已成为日常使用的技术（从 Android 智能手机到超级计算机）背后的无形力量，并为全球相当大一部分服务器提供动力。[^8] 当托瓦兹在 1991 年发布 Linux 内核时，大多数操作系统都是专有的且价格昂贵。他创建了一个任何人都可以使用、修改和共享的免费操作系统，重新定义了软件的可能性，并点燃了全球开源革命。[^8]

由于托瓦兹的工作，互联网变得更加开放、安全和创新，开源模式鼓励了知识共享并加速了技术进步。[^8] Linux 是云计算平台的基础，运行着大多数 Web 服务器，驱动着世界上最快的超级计算机，并作为 Android——全球使用最广泛的移动操作系统——的基础。这种普遍性意味着数十亿人每天都在接触托瓦兹的作品，而且常常不知道。

### 塑造开源文化

除了技术成就，托瓦兹在塑造开源社区、促进透明度、合作和社区驱动的创新方面发挥了重要作用。[^8] 他对开源原则的承诺一直是 Linux 成功的驱动力，创造了一个让全球程序员能够为操作系统改进和发展做出贡献的环境。[^8]

托瓦兹将 Linux 作为开源发布的决定不仅仅启动了一个运动——它定义了我们今天受益的开源文化。[^8] 他经常将开源与科学进行比较：“开源允许人们在前人知识的基础上发展，而不会被愚蠢地隐藏。”[^7] 这种理念使得前所未有的规模的协作开发成为可能，证明了大型、复杂的软件可以通过分布式协作而不是传统的公司或层级结构来创建。

### Git 的持久影响

Git 已成为全球协作软件开发的关键工具。[^8] 托瓦兹于 2005 年创建的分布式版本控制系统彻底改变了开发人员管理和跟踪代码变化的方式。[^8] Git 的影响力远远超出了 Linux——它已成为软件开发中版本控制的标准，使得协作编码实践成为现代软件工程的标志。

基于 Git 的平台，尤其是 GitHub，为软件协作、开源贡献甚至文档和写作等非软件项目创造了全新的范式。Git 的设计体现了托瓦兹的工程理念：实用、可靠且专注于高效地解决实际问题。

### 文化和经济影响

托瓦兹工作的经济影响是无法估量的。从初创公司到谷歌、亚马逊和 Facebook 等科技巨头，公司都依靠 Linux 来运营其基础设施。使用免费开源软件而非专有替代品所节省的成本，促进了前所未有的创新和创业。

托瓦兹的遗产超越了代码，它体现了持续塑造全球软件开发和分发方式的协作创新精神。[^8] 他的工作证明了去中心化的、志愿者驱动的开发能够创建在质量和可靠性上与专有软件相媲美甚至超越的软件。这个概念验证激发了无数其他开源项目，并从根本上改变了关于如何创造有价值软件的假设。

### 持续的影响

55 岁的托瓦兹仍然积极参与 Linux 内核的开发，审查代码并对哪些内容将被纳入做出最终决定。他的持续参与确保了项目方向的稳定性和一致性，尽管这也引发了关于继承计划以及如此关键的全球基础设施的长期治理的问题。

## 名言

> "废话少说。给我看代码。" - 强调托瓦兹务实、以行动为导向的哲学，重视工作实现而非抽象讨论[^7]

> "我不是一个富有远见的人。我是一名工程师。我乐于那些四处张望看星星的人，但我正在观察地面，我想在摔倒之前修好那个坑。" - 描述他务实、解决问题的软件开发方法，而非追求宏伟的愿景[^7]

> "大多数优秀的程序员写程序不是因为他们期望得到报酬或公众的赞扬，而是因为编程很有趣。" - 阐述了他对软件开发中内在动力的看法[^7]

> "在真正的开源中，你有权控制自己的命运。" - 解释了开源软件为用户和开发者提供的基本自由[^7]

> "我经常将开源与科学进行比较。就像科学把发展思想、改进他人想法的整个概念公开化，并将其发展成今天的科学，以及我们取得的不可思议的进步一样。" - 将开源开发与科学进步进行类比[^7]

> "开源最重要的部分是人们可以做他们擅长的事情。" - 强调开源如何让贡献者专注于自己的优势[^7]

> "我不是一个好人，我不在乎你。我关心的是技术和内核——那对我来说才是重要的。" - 在一次会议上的争议性声明，体现了他专注于技术卓越而非人际关系的特点[^5]

> "Linux 的哲学是‘嘲笑危险’。糟糕。错了。‘自己动手’。是的，就是这个。" - 有趣地描述了 Linux 开发背后的自力更生的精神[^7]
