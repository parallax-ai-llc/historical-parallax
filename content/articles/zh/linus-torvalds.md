## 摘要

林纳斯·本尼迪克特·托瓦兹（Linus Benedict Torvalds）是一位芬兰裔美国软件工程师，他创造并持续领导着 Linux 内核的开发，Linux 内核是 Linux 操作系统的基础。托瓦兹于 1969 年出生在赫尔辛基，1991 年作为一名大学生成为 Linux 的开发者，由此引发了一场改变科技格局的全球开源革命。他还创造了 Git，这是一个革命性的分布式版本控制系统，彻底改变了协作软件开发。他务实、精英主义的软件开发方法使他成为计算机史上最具影响力的人物之一，尽管他粗鲁的沟通方式也引发了显著的争议。

## 早年生活和背景

林纳斯·本尼迪克特·托瓦兹于 1969 年 12 月 28 日出生在芬兰赫尔辛基，父母是记者安娜（Anna）和尼尔斯·托瓦兹（Nils Torvalds）。[^1] 他的家庭属于瑞典语系少数民族，约占芬兰总人口的 5%。[^1] 托瓦兹的名字来源于美国诺贝尔奖得主化学家林纳斯·鲍林（Linus Pauling）。他出身于一个知识渊博的家庭——他的祖父利奥·托恩奎斯特（Leo Törnqvist）是统计学家，另一位祖父奥利·托瓦兹（Ole Torvalds）是诗人。[^1][^2]

托瓦兹的父母在 20 世纪 60 年代是赫尔辛基大学的校园激进分子，营造了一种重视思想独立和批判性思维的家庭氛围。[^1] 他与计算机结缘始于 20 世纪 80 年代初，当时他的祖父购买了一台 Commodore VIC-20 家用电脑。这次购买至关重要——11 岁的托瓦兹对编程产生了浓厚的兴趣，花费无数时间编写代码并探索这台机器的功能。[^1][^2]

年轻的托瓦兹表现出卓越的编程天赋，经常将计算机工作置于社交活动之上。早期沉浸于计算机领域，为他后来的职业生涯奠定了基础：专注于实际问题解决、动手实验以及优先处理与机器打交道而非人际交往。他在芬兰瑞典语社区的童年也让他拥有了多元文化的视角，在以芬兰语为主的国家里，他从小就流利掌握瑞典语。

结合了知识分子的家庭传承、早期接触计算技术以及对技术挑战的个人倾向，为托瓦兹后来的成就奠定了基础。他祖父对 Commodore VIC-20 的投资，无意中启动了一系列将重塑全球计算的事件。

## 声名鹊起

托瓦兹于 1988 年至 1996 年就读于赫尔辛基大学，并在 NODES 研究小组攻读计算机科学硕士学位。[^1] 他的大学学业在第一年后中断，因为他于 1989 年夏天加入了芬兰海军兰德陆军旅，以履行芬兰的强制兵役。托瓦兹选择了为期 11 个月的军官培训项目，并获得了少尉军衔，担任炮兵观察员。[^1]

1990 年回到大学后，托瓦兹第一次接触到了 Unix，当时他使用的是一台运行着 ULTRIX 操作系统的 DEC MicroVAX 计算机。这次接触具有划时代的意义。[^1] 他购买了计算机科学教授安德鲁·塔能鲍姆（Andrew Tanenbaum）的著作《操作系统：设计与实现》，该书讨论了 MINIX，一个为教学目的设计的小型类 Unix 操作系统。1991 年 1 月 5 日，托瓦兹购买了一台基于 Intel 80386 的 IBM PC 克隆机，在收到一份 MINIX 副本后，他开始了他将成为 Linux 的项目。[^1][^3]

从 1991 年 4 月到 8 月，21 岁的托瓦兹编写了最初的 Linux 内核。[^3] 他从 Intel 80386 汇编语言的任务切换器和终端驱动程序开始，专门为他的硬件编写程序，独立于一个完整的操作系统。[^3] 1991 年 8 月 25 日，托瓦兹在 comp.os.minix 新闻组上发布了他著名的公告，将他的项目描述为一个面向 386/486 计算机的业余操作系统。[^3]

1991 年 9 月 17 日，托瓦兹准备了 Linux 的 0.01 版本，并将其上传到芬兰大学和研究网络（FUNET）的 FTP 服务器。[^3] 这个早期版本包含 10,239 行代码。[^3] 1991 年 10 月 5 日，他宣布了第一个“官方”版本 0.02。[^3] 关键的突破发生在 1991 年 12 月，推出了 0.11 版本，这是第一个能够自我托管的版本——即可以在运行 Linux 内核的计算机上进行编译。[^3]

有趣的是，托瓦兹最初想将他的创作命名为“Freax”，这是“free”（自由）、“freak”（怪胎）和“x”（指代 Unix）的混合词。他曾考虑过“Linux”，但认为这个名字过于自大而放弃了。[^3] 赫尔辛基理工大学的 FTP 服务器管理员之一阿里·莱姆克（Ari Lemmke）不喜欢“Freax”，在没有咨询托瓦兹的情况下，将项目命名为“Linux”。然而，后来托瓦兹同意了这个名字。[^3]

## 主要成就

### Linux 内核

托瓦兹的主要成就是自 1991 年以来创建和维护 Linux 内核。[^8] 这个最初只是一个学生项目，后来发展成为一种操作系统，驱动着世界相当一部分计算基础设施。Linux 已成为互联网的支柱，运行在全球的服务器上，并作为 Android 智能手机、超级计算机、嵌入式系统和无数其他设备的操作系统。[^8]

Linux 的成功不仅源于其技术优势，还源于托瓦兹将其开源发布的决定。这一选择不仅仅是启动了一个项目——它催生了现代开源运动。[^8] 通过允许任何人自由地使用、修改和分发代码，托瓦兹建立了一种前所未有的协作开发模式。全球成千上万的开发者为 Linux 贡献了力量，创造了人类历史上最大的协作项目之一。

托瓦兹的发展理念强调实用主义而非意识形态。正如他所说，开源“不是一场十字军东征——它只是一种更好的合作和生成代码的方式。”[^7] 这种实用的方法吸引了那些专注于解决实际问题而非追求抽象原则的贡献者。他坚持认为，“开源最重要的部分是人们可以做他们擅长的事情。”[^7]

### Git 的创造

2005 年，托瓦兹在创造分布式版本控制系统 Git 时做出了另一项开创性的贡献。[^4] Linux 内核开发社区一直在使用 BitKeeper，这是一个专有工具，但随着与商业开发商的关系破裂，托瓦兹需要一个替代品。[^4]

托瓦兹没有采用现有系统，而是仅用十天时间就编写了 Git。[^4] 他在 2005 年 4 月 7 日提交了 Git 的第一个提交。[^4] 在项目开始的几天内，Git 就完全实现了自我托管，并在几周内准备好用于 Linux 内核开发。[^4] 托瓦兹的主要目标是分发、高性能和绝对的可靠性，以捕获任何数据损坏。[^4]

Git 彻底改变了全球的软件开发。托瓦兹维护了 Git 大约四个月，然后将其维护权移交给了从开发第一周就参与其中的 Junio Hamano。[^4] 如今，Git 已成为占主导地位的版本控制系统，被全球数百万开发者和组织使用。GitHub、GitLab 和 Bitbucket 等平台围绕 Git 构建了整个生态系统，从根本上改变了软件的开发和共享方式。

### 奖项和认可

托瓦兹的贡献获得了广泛认可。他于 2012 年获得“千年技术奖”，以表彰他“创造了一个新的计算机开源操作系统，并由此诞生了被广泛使用的 Linux 内核。”[^6] 其他主要奖项包括 2014 年 IEEE 计算机学会计算机先驱奖、2018 年 IEEE 曾真消费者电子产品奖、NEC C&C 奖（2010 年）、武田社会/经济福祉奖（2008 年）、英国计算机学会的洛夫莱斯奖章（2000 年）和电子前哨基金会先驱奖（1998 年）。[^6]

他于 2012 年入选互联网名人堂，并于 2008 年入选计算机历史博物馆名人堂。[^6] 《时代》杂志在 2000 年的“时代 100：世纪最重要人物”投票中将他列为第 17 位，并在 2004 年被评为世界上最具影响力的人物之一。[^6]

### 其他贡献

除了 Linux 和 Git，托瓦兹还从 2011 年底开始创建了 Subsurface，这是一个用于记录和规划潜水活动的软件。[^4] 尽管这个项目不如他的其他作品出名，但它展示了他持续的动手软件开发方法以及为满足个人需求而构建工具的意愿。

## 争议和批评

### 粗鲁的沟通风格

托瓦兹的管理风格在他的职业生涯中引发了大量的争议。他在 Linux 内核邮件列表（LKML）上以其粗鲁而闻名，经常对他认为有问题甚至人进行恶毒的、充满脏话的辱骂。[^5] 在一次会议上，托瓦兹表示：“我不是一个好人，我不在乎你。我在乎技术和内核——这对我来说很重要。”[^5]

他严厉的沟通方式给许多开发者营造了有毒的氛围。著名的 Linux 开发者莎拉·夏普（Sarah Sharp）公开批评托瓦兹的“污言秽语和残酷的贬低”，并警告说她“再也受不了了”，并恳求道：“你不需要大喊大叫，骂我名字，或者叫我闭嘴！”[^5] 另一位开发者指出：“他在新人面前斥责资深开发者。这不太可能鼓励人们想要成为资深开发者。”[^5]

托瓦兹所培养的文化超越了他个人——Linux 内核的维护者也常常变得直率、粗鲁或野蛮，顶级开发者之间经常互相大喊来纠正行为。[^5] 虽然有些人辩称这是为了维持高标准，但另一些人则认为这增加了进入门槛，并赶走了那些觉得环境不友好的有才华的贡献者。

### 2018 年道歉和行为准则

2018 年 9 月中旬，托瓦兹宣布他将休假一段时间，以“寻求帮助，以了解他人的情感并做出适当的反应。”[^5] 这个前所未有的举动是在多年来对其行为的批评不断累积之后发生的。与此同时，Linux 内核项目采用了新的行为准则，旨在使社区更具包容性，并消除侮辱、贬低性的评论和行为。[^5]

行为准则取代了 2015 年的“冲突准则”，该准则仅建议人们“善待彼此”。[^5] 然而，新的准则引发了巨大的反对。一些程序员认为这是对精英主义的攻击，而另一些人则称这些攻击是影射的厌女症。[^5] 一些人担心该准则可能会因缺乏政治正确性而导致他们被取消内核开发者的资格，而另一些人则认为该准则缺乏足够的执行机制来带来有意义的改变。[^5]

### 对多样性和包容性的影响

托瓦兹的攻击性风格以及由此产生的文化因其阻碍内核开发多样性而受到批评。Linux 内核社区历史上绝大多数是男性，主要是白人和亚洲人，女性和代表性不足的少数族裔的参与有限。批评者认为，托瓦兹确立并容忍的敌对沟通规范，极大地导致了这种缺乏多样性的状况。

这场争议凸显了不同精英主义观念之间的紧张关系：一种只关注代码质量，而不顾人们如何被对待；另一种则认为包容性的环境是获取最广泛人才库的必要条件。托瓦兹最终承认他需要改变，这表明他认识到他以前的方法是有代价的，尽管人们对其后续改变是否充分存在不同意见。

## 个人生活

林纳斯·托瓦兹于 20 世纪 90 年代中期与曾六次获得芬兰空手道全国冠军的托芙·托瓦兹（Tove Torvalds，娘家姓 Monni）结婚。[^9] 他们第一次相遇是在 1993 年秋天，当时托瓦兹在赫尔辛基大学为学生们举办入门级计算机实验室课程。他指示课程参与者给他发一封电子邮件作为测试，托芙回复了一封电子邮件，邀请他约会。[^9]

这对夫妇有三个女儿：帕特里夏·米兰达（Patricia Miranda，1996 年出生）、丹妮拉·尤兰达（Daniela Yolanda，1998 年出生）和塞莱斯特·阿曼达（Celeste Amanda，2000 年出生），其中两人出生在美国。[^9] 在一次典型的技术个性展示中，Linux 内核的重启系统调用接受了她们的出生日期（以十六进制书写）作为魔术值——这是永久嵌入代码中的一个异想天开的致敬。[^9]

托瓦兹于 1997 年移居美国，并与家人定居。[^9] 他于 2010 年成为美国公民，同时保留芬兰国籍。[^9] 他的家人一直居住在俄勒冈州波特兰市，托瓦兹在那里继续他的 Linux 内核工作。

除了编程，托瓦兹还对潜水表现出兴趣，这促使他开发了 Subsurface 潜水日志软件。[^4] 这个爱好展示了他通过软件开发解决个人需求的倾向——当现有的潜水日志软件不符合他的要求时，他自己构建了一个。

托瓦兹将自己描述为务实而非富有远见：“我不是一个有远见的人。我是一个工程师。我喜欢那些漫步仰望星空的人，但我关注的是脚下，我想在掉进坑里之前把它修好。”[^7] 这种自我评价反映了他务实的、解决问题的导向，以及他对具体改进而非抽象理论的偏好。

他的个性结合了对技术卓越的强烈关注，以及对处理社交和情感动态的明显不适。尽管他在 2018 年的争议后努力改善了他的沟通方式，但同事们指出，他的基本方向仍然是关注代码质量和技术功绩，而不是人际关系。

## 遗产

### 计算机基础设施的转型

林纳斯·托瓦兹对全球计算基础设施的影响怎么强调都不为过。Linux 已成为日常技术背后的无形力量，从 Android 智能手机到超级计算机，并驱动着全球相当一部分服务器。[^8] 当托瓦兹于 1991 年发布 Linux 内核时，大多数操作系统都是专有的且价格昂贵。他创建了一个任何人都可以使用、修改和共享的免费操作系统，重新定义了软件的可能性，并引发了全球开源革命。[^8]

由于托瓦兹的工作，网络变得更加开放、安全和创新，而开源模式鼓励了知识共享并加速了技术发展。[^8] Linux 是云计算平台的基础，运行着大部分 Web 服务器，驱动着世界上最快的超级计算机，并作为 Android——全球使用最广泛的移动操作系统——的基础。这种普及意味着数十亿人每天都在与托瓦兹的作品互动，而他们通常并不知道。

### 塑造开源文化

除了技术成就，托瓦兹还在塑造开源社区、促进透明度、合作和社区驱动的创新方面发挥了至关重要的作用。[^8] 他对开源原则的承诺一直是 Linux 成功的驱动力，创造了一个让全球程序员能够为操作系统改进和发展做出贡献的环境。[^8]

托瓦兹将 Linux 开源发布的决定不仅启动了一个运动——它定义了我们今天受益的开源文化。[^8] 他经常将开源与科学相提并论：“开源允许人们在坚实的前人知识基础上进行开发，而没有愚蠢的隐藏。”[^7] 这种理念使得前所未有的大规模协作开发成为可能，证明了大型、复杂的软件可以通过分布式协作而无需传统的公司或层级结构来创建。

### Git 的持久影响

Git 已成为全球协作软件开发的核心工具。[^8] 托瓦兹于 2005 年创建的分布式版本控制系统彻底改变了开发者管理和跟踪代码变更的方式。[^8] Git 的影响远远超出了 Linux——它已成为软件开发中版本控制的标准， enabling the collaborative coding practices that define modern software engineering.

基于 Git 的平台，尤其是 GitHub，为软件协作、开源贡献甚至非软件项目（如文档和写作）创造了全新的范式。Git 的设计体现了托瓦兹的工程理念：实用、可靠，并专注于高效地解决实际问题。

### 文化和经济影响

托瓦兹工作的经济影响是无法估量的。从初创公司到谷歌、亚马逊和 Facebook 等科技巨头，各类公司都依赖 Linux 来支撑其基础设施。使用免费开源软件而非专有替代品所节省的成本，使得前所未有的创新和创业成为可能。

托瓦兹的遗产超越了代码，体现了持续塑造全球软件开发和分发方式的协作创新精神。[^8] 他的工作证明了去中心化的、志愿者驱动的开发可以创造出在质量和可靠性上能与专有替代品相媲美甚至超越的软件。这个概念验证激发了无数其他开源项目，并从根本上改变了关于如何创造有价值的软件的假设。

### 持续的影响

55 岁的托瓦兹仍然积极参与 Linux 内核的开发，审查代码并最终决定哪些内容被整合。他的持续参与确保了项目方向的稳定性和一致性，尽管这也引发了关于继任计划以及如此关键的全球基础设施的长期治理问题。

## 名言

> “说空话没有用。给我看代码。” - 强调托瓦兹务实、注重行动的哲学，重视实际实现而非抽象讨论[^7]

> “我不是一个有远见的人。我是一个工程师。我喜欢那些漫步仰望星空的人，但我关注的是脚下，我想在掉进坑里之前把它修好。” - 描述他务实的、解决问题的软件开发方法，而非追求宏大的愿景[^7]

> “大多数优秀的程序员写程序不是因为他们期望获得报酬或公众的赞扬，而是因为编程很有趣。” - 阐述了他对软件开发中内在动力的看法[^7]

> “在真正的开源中，你有权控制自己的命运。” - 解释了开源软件为用户和开发者提供的基本自由[^7]

> “我经常将开源与科学相提并论。就像科学把这种在开放环境中发展想法、在他人想法的基础上进行改进的概念发展到今天的科学，以及我们取得的惊人进步一样。” - 将开源开发与科学进步进行类比[^7]

> “开源最重要的部分是人们可以做他们擅长的事情。” - 强调开源如何使贡献者能够专注于自己的优势[^7]

> “我不是一个好人，我不在乎你。我在乎技术和内核——这对我来说很重要。” - 在一次会议上的争议性发言，体现了他专注于技术卓越而非人际关系[^5]

> “Linux 的哲学是‘嘲笑危险’。糟糕。错了。‘自己动手’。是的，就是这样。” - 对 Linux 开发背后自力更生的精神进行的幽默描述[^7]
